# 员工端我的统计培训 - 部署文档

## 一、项目概述

本项目是员工端个人中心统计培训系统，包含以下功能模块：
- 员工个人信息展示
- 任务记录统计
- 培训资料学习
- 安全设置管理

**技术栈：**
- 前端：Vue 3 + Vue Router + Vite
- 后端：Node.js + Express
- 端口：前端 5173，后端 3001

## 六、登录态机制

### 6.1 登录流程

1. 用户在登录页输入工号和密码
2. 前端调用登录接口 `/api/employee/login`
3. 后端验证通过后生成 Token 并返回
4. 前端将 Token 和员工信息存储到 localStorage
5. 后续请求在请求头中携带 Token

### 6.2 Token 存储

**存储位置：** `localStorage`

| 键名 | 说明 |
|------|------|
| `token` | 登录认证令牌 |
| `employeeInfo` | 员工基本信息（JSON 字符串） |

**登录实现：** [Login.vue](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/020-员工端我的统计培训/frontend/src/components/Login.vue#L64-L87)

```javascript
const handleLogin = async () => {
  // ...
  const result = await api.auth.login({ employeeId, password }, router);
  if (result && result.code === 0) {
    localStorage.setItem('token', result.data.token);
    localStorage.setItem('employeeInfo', JSON.stringify(result.data.employee));
    router.push('/');
  }
  // ...
};
```

### 6.3 Token 校验机制

**前端路由守卫：** [router.js](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/020-员工端我的统计培训/frontend/src/router.js#L21-L32)

```javascript
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token');
  const requiresAuth = to.meta.requiresAuth !== false;

  if (requiresAuth && !token) {
    next({ path: '/login' });
  } else if (to.path === '/login' && token) {
    next({ path: '/' });
  } else {
    next();
  }
});
```

**请求头携带 Token：** [api.js](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/020-员工端我的统计培训/frontend/src/utils/api.js#L3-L9)

```javascript
function getAuthHeaders() {
  const token = localStorage.getItem('token');
  return {
    'Content-Type': 'application/json',
    'Authorization': token ? `Bearer ${token}` : ''
  };
}
```

**后端认证中间件：** [auth.js](file:///Users/wuzhijie/Documents/xiaohongshu/biaozhu/yxz-trae-projects/020-员工端我的统计培训/backend/middleware/auth.js#L9-L32)

```javascript
function authenticateToken(req, res, next) {
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.startsWith('Bearer ')
    ? authHeader.slice(7)
    : req.headers['x-token'];

  if (!token) {
    return res.status(401).json({ code: -1, message: '未登录，请先登录' });
  }

  const employeeId = activeTokens.get(token);
  if (!employeeId) {
    return res.status(401).json({ code: -1, message: '登录已过期，请重新登录' });
  }
  // ...
}
```

### 7.5 一键验收脚本

创建 `验收脚本.sh`：

```bash
#!/bin/bash

echo "=========================================="
echo "员工端我的统计培训 - 验收脚本"
echo "=========================================="

# 颜色定义
RED='\033[0;31m'
GREEN='\033[0;32m'
NC='\033[0m'

# 1. 检查后端服务
echo -e "\n[1/5] 检查后端服务..."
if curl -s http://localhost:3001/api/employee/login -X POST \
  -H "Content-Type: application/json" \
  -d '{"employeeId":"EMP001","password":"123456"}' | grep -q "success"; then
  echo -e "${GREEN}✓ 后端服务正常${NC}"
else
  echo -e "${RED}✗ 后端服务异常${NC}"
  exit 1
fi

# 2. 登录测试
echo -e "\n[2/5] 登录接口测试..."
LOGIN_RESPONSE=$(curl -s -X POST http://localhost:3001/api/employee/login \
  -H "Content-Type: application/json" \
  -d '{"employeeId":"EMP001","password":"123456"}')

if echo "$LOGIN_RESPONSE" | grep -q '"code":0'; then
  TOKEN=$(echo "$LOGIN_RESPONSE" | grep -o '"token":"[^"]*"' | cut -d'"' -f4)
  echo -e "${GREEN}✓ 登录成功，Token: ${TOKEN:0:20}...${NC}"
else
  echo -e "${RED}✗ 登录失败${NC}"
  exit 1
fi

# 3. 培训资料接口测试
echo -e "\n[3/5] 培训资料接口测试..."
TRAINING_RESPONSE=$(curl -s -H "Authorization: Bearer $TOKEN" \
  http://localhost:3001/api/employee/training)

if echo "$TRAINING_RESPONSE" | grep -q '"code":0'; then
  COUNT=$(echo "$TRAINING_RESPONSE" | grep -o '"total":[0-9]*' | cut -d':' -f2)
  echo -e "${GREEN}✓ 培训资料接口正常，共 ${COUNT} 条资料${NC}"
else
  echo -e "${RED}✗ 培训资料接口异常${NC}"
fi

# 4. 培训统计接口测试
echo -e "\n[4/5] 培训统计接口测试..."
STATS_RESPONSE=$(curl -s -H "Authorization: Bearer $TOKEN" \
  http://localhost:3001/api/employee/training/stats)

if echo "$STATS_RESPONSE" | grep -q '"code":0'; then
  echo -e "${GREEN}✓ 培训统计接口正常${NC}"
else
  echo -e "${RED}✗ 培训统计接口异常${NC}"
fi

# 5. 前端页面访问测试
echo -e "\n[5/5] 前端页面访问测试..."
if curl -s http://localhost:5173 | grep -q "<!DOCTYPE html>"; then
  echo -e "${GREEN}✓ 前端页面可访问${NC}"
else
  echo -e "${RED}✗ 前端页面异常${NC}"
fi

echo -e "\n=========================================="
echo -e "${GREEN}所有验收项通过！${NC}"
echo "=========================================="
```

## 八、常见问题

### 8.1 端口被占用

```bash
# 查找占用端口的进程
lsof -ti:3001 | xargs kill -9  # 释放后端端口
lsof -ti:5173 | xargs kill -9  # 释放前端端口
```

### 8.2 跨域问题

确认 Vite 代理配置正确，前端请求使用 `/api` 前缀，不要直接使用 `http://localhost:3001`。

### 8.3 Token 无效

清除浏览器 localStorage 中的 token，重新登录：

```javascript
localStorage.removeItem('token');
localStorage.removeItem('employeeInfo');
```

### 8.4 依赖安装失败

```bash
# 清理缓存后重新安装
rm -rf node_modules package-lock.json
npm cache clean --force
npm install
```
